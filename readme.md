## 1. Overview

This repository contains all code required to reproduce the numerical experiments, tables, and figures in the paper *The Data-Driven Censored Newsvendor Problem*. The codebase implements learning algorithms for censored-demand newsvendor problems, evaluates their performance on synthetic and real-world data, and produces all figures and tables reported in the manuscript.

A typical replication workflow is:
1. Set up the computational environment (Section 4).
2. Run simulation scripts to generate synthetic or real-world results (Section 5).
3. Run plotting and table-generation scripts to reproduce all figures and tables.

---

## 2. Data availability and provenance

This paper relies on both **synthetically generated data** and **real-world retail data**.

### Synthetic data
All synthetic datasets are generated by the authors using the scripts provided in this repository. No external data sources are required. The data-generating processes (including demand distributions, censoring mechanisms, and robustness parameters) are fully specified in the simulation scripts (e.g., `run_synthetic.py`, `run_well_separated.py`, `run_vanilla_regret.py`). Running these scripts reproduces the synthetic datasets used in the paper.

### Real-world data
The real-world experiments use retail sales data derived from the [FreshRetailNet Dataset](https://huggingface.co/datasets/Dingdong-Inc/FreshRetailNet-50K). These data are not redistributed here in raw form. The repository assumes the presence of the original files (located in `./freshnet_data_sims/datasets/train.parquet`, `./freshnet_data_sims/datasets/eval.parquet`).  To process the dataset and convert to a csv run `./freshnet_data_Sims/datasets/initial_data_process.py` to generate `.csv` files from the code.

## 3. Variable dictionaries

Below we describe the key variables used across datasets and scripts.

### Common variables
- `D`: true (latent) demand
- `y`: observed (censored) demand, `y = min(D, q)`
- `q`: order quantity
- `qbar`: maximum allowable order quantity (used $M$ in the manuscript)
- `b`: underage (stockout) cost
- `h`: holding cost (fixed to 1 in all experiments)
- `rho`: critical fractile, `rho = b / (b + h)`
- `lam`: robustness / censoring parameter
- `N`: number of historical observations
- `metric`: performance metric being recorded
- `value`: numerical value of the metric

### Metrics
- `true_cost`: expected newsvendor cost under the true demand distribution
- `minmax_cost`: robust min-max cost under the ambiguity set
- `id`: identifiability label (`identifiable`, `knife_edge`, `unidentifiable`)

## 4. Computational requirements

All code is written in **Python (>= 3.9)**.

Key dependencies include:
- numpy
- pandas
- scipy
- matplotlib
- seaborn
- lifelines
- statsmodels

We provide an **`environment.yml`** file specifying all required packages and versions to fully reproduce the computational environment. This environment can be created using:
```bash
conda env create -f environment.yml
conda activate robust_inventory
```

All random number generation uses fixed seeds where appropriate to ensure reproducibility. Some Monte Carlo benchmarks use large sample sizes (up to 1e7), which may require several minutes to run on a standard laptop.

## 5. Programs / Code

### Simulation scripts
- `run_synthetic.py` — Runs simulations on synthetically generated demand data.
- `run_well_separated.py` — Runs simulations for the well-separated synthetic demand setting.
- `run_vanilla_regret.py` — Runs simulations evaluating vanilla regret relative to the true optimum.
- `run_synthetic_knife_edge.py` - Runs simulations evaluating robust and vanilla regret in the knife-edge regime.
- `freshnet_data_sims/datasets/initial_data_process.py` — Generates the processed dataset for the real-world simulations.
- `freshnet_data_sims/run_real.py` — Runs simulations on real-world retail data.
- `freshnet_data_sims/run_identifiable.py` — Runs identifiability tests on real-world retail data.

### Figure and table generation
- `delta_qcrit_figure.py` — Plots $q^{\Delta}$ and $\Delta$ as functions of $\lambda$ for well-separated distributions.
- `minimaxrisk_figure.py` — Plots minimax regret as a function of the order level.
- `normal_std_plot.py` — Plots CDFs of the Gaussian distributions used in the simulations.
- `synthetic_lam_tables.py` — Creates tables comparing algorithm performance across $\lambda$.
- `synthetic_var_tables.py` — Creates tables comparing performance under varying demand variance.
- `synthetic_vanilla_lam_tables.py` — Creates tables comparing algorithm performance on vanilla regret across $\lambda$.
- `synthetic_well_separated_lam_tables.py` — Creates tables comparing algorithm performance on regret for well-separated distributions across $\lambda$.
- `synthetic_plots.py` — Plots algorithm performance as a function of sample size $N$.
- `synthetic_knife_edge_tables.py` - Plots worst-case algorithm performance for knife-edge setting.
- `synthetic_knife_edge_vanilla_tables.py` - Plots vanilla regret algorithm performance in the knife-edge setting.
- `freshnet_data_sims/plot_algo_performance.py` — Produces figures summarizing algorithm performance on real-world data.
- `freshnet_data_sims/plot_dataset_stats.py` — Generates descriptive statistics and dataset diagnostics.

All scripts use relative paths and can be run independently once the required data and environment are in place. Running the simulation scripts followed by the plotting/table scripts reproduces all main results in the paper.
